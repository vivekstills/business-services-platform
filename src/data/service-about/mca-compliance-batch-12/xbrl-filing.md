# XBRL Filing

## Detailed Overview

XBRL (eXtensible Business Reporting Language) is a global standard for digital business reporting that enables structured, machine-readable financial data. Under the Companies Act, 2013 and the Companies (Filing of Documents and Forms in XBRL) Rules, 2015, certain categories of companies are required to file their financial statements (AOC-4) and Cost Audit Reports in XBRL format on the MCA portal.

XBRL filing involves tagging each financial data element using a standardised taxonomy — enabling regulators, analysts, and stakeholders to extract, compare, and analyse company financials efficiently across thousands of filings.

## Applicability

XBRL filing is mandatory for:

| Category | Trigger |
| --- | --- |
| Listed companies | All listed companies on Indian stock exchanges |
| Subsidiaries of listed companies | All Indian subsidiaries |
| Companies with paid-up capital ≥ ₹5 crore | Mandatory |
| Companies with turnover ≥ ₹100 crore | Mandatory |
| Companies preparing CFS | Required for consolidated filings |
| Cost audit applicable companies | Cost Audit Report filed in XBRL via Form CRA-4 |

Applicable pan-India.

## Compliance Requirements

| Requirement | Detail |
| --- | --- |
| AOC-4 XBRL | Annual financial statements |
| CRA-4 XBRL | Cost audit reports |
| Taxonomy | MCA prescribed XBRL taxonomy (updated periodically) |
| Tagging accuracy | Each financial element correctly tagged |
| Validation | XBRL file must pass MCA validation tool |
| Professional certification | CA/CS/CMA certification mandatory |
| Software | XBRL preparation software (multiple vendors) |

## Benefits

- Standardised financial reporting
- Data interoperability for analysts and regulators
- Reduced manual data extraction
- Improved data quality and consistency
- Transparency for investors

## Risks / Penalties

| Default | Penalty |
| --- | --- |
| Late filing | ₹100 per day (additional) |
| Tagging errors | Form rejection requiring resubmission |
| Validation failures | Cannot upload — must correct XBRL file |
| Wrong taxonomy version | Form rejection |
| Non-compliance with XBRL where mandatory | AOC-4 in non-XBRL format will be rejected |

## Documents Required

Same as AOC-4 (audited financials, board report, auditor report) but converted to XBRL format using approved software and MCA taxonomy.

## Process / Procedure

1. Step 1 — Complete audit of financial statements.
2. Step 2 — Engage XBRL preparation expert/software.
3. Step 3 — Tag each financial data element using current MCA taxonomy.
4. Step 4 — Generate XBRL instance document.
5. Step 5 — Validate using MCA validation tool.
6. Step 6 — Get CA/CS/CMA certification.
7. Step 7 — File AOC-4 XBRL on MCA portal.

## Government Fees

Same as AOC-4 — based on authorized capital. XBRL preparation software costs and professional fees are additional.

## Timeline

Same as AOC-4 — within 30 days of AGM.

## Frequently Asked Questions

1. **What is the MCA XBRL taxonomy and how often does it change?** The MCA XBRL taxonomy is the standardised set of tags used to mark up financial data. MCA updates the taxonomy periodically (typically annually) to reflect changes in accounting standards (Ind AS) and disclosure requirements. The latest taxonomy version must be used for the relevant FY.
2. **Can XBRL filing be done without specialised software?** Practically no. Manual XBRL preparation is extremely complex. Professional XBRL software (from vendors like IRIS Carbon, BSE XBRL, Thomson Reuters) is essential. Many CA/CS firms also provide XBRL filing services.
3. **What's the difference between AOC-4 and AOC-4 XBRL?** AOC-4 is the standard PDF-based filing. AOC-4 XBRL is the same financial information in machine-readable XBRL format with tagged data elements. XBRL enables automated data extraction; PDF is human-readable only.
4. **Is XBRL filing required for cost audit reports?** Yes. Companies subject to cost audit under Section 148 of the Companies Act must file Cost Audit Report (CRA-3) in Form CRA-4 in XBRL format.
5. **Are subsidiaries of listed companies required to file XBRL even if they are small?** Yes. All subsidiaries of listed companies must file in XBRL format, regardless of their individual size or paid-up capital. The trigger is the parent's listed status, not the subsidiary's individual size.
