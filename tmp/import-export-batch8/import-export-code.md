# Import Export Code (IEC) registration

## Overview

**Import Export Code (IEC)** is a 10-digit identifier issued by the **Directorate General of Foreign Trade (DGFT)**, Ministry of Commerce. It is the primary authorisation to carry on **import** and **export** of **goods** and **services** in India, under the **Foreign Trade (Development and Regulation) Act, 1992** and the **Foreign Trade Policy**. From **2021** integration with **PAN**, the **IEC** is typically the same as the entity **PAN** for a single unified identity. There is no expiry of the code itself, but **annual IEC update** on the **DGFT** portal is **mandatory**; failure deactivates **IEC** and blocks trade, banks, and schemes.

**IEC** is checked by **Customs** (**BoE** / **SB**), **banks** for trade remittances, **DGFT** for licences and incentives, and **export promotion councils** for **RCMC**. **Legal** import or export of goods, in the usual path, is not possible without a valid **active** **IEC** where it is required.

**Common exemptions (illustrations only):** personal non-commercial imports/exports, certain **government** departments, baggage rules, and specific **FTP** carve-outs, each as per **notification** in force.

## Applicability

| Situation | IEC |
|-----------|-----|
| Company / firm **importing** or **exporting** **goods** | **Mandatory** (usual path) |
| **Commercial** **import** or **export** by proprietor / individual | **Mandatory** where law applies |
| **LLP** / **partnership** in **EXIM** | **Mandatory** |
| **Trust** / **society** in trade | **Mandatory** if not exempted |
| **E-commerce** **goods** to overseas buyers | **Mandatory** in typical marketplace onboarding |
| **Service** **exporters** (registered entity), **FEMA** / **STPI**-linked where applicable | **Mandatory** in typical firm setups |
| **RCMC**, **Advance Authorisation**, **EPCG**, **RoDTEP** | **Prerequisite** **IEC** in usual path |

**Not required (examples, subject to fact and law):** non-trade personal movements, some **government** transactions, and goods exempt under a specific **FTP** line.

**One** **IEC** per **PAN**-entity, **all-India**; one code for all ports and products for that **legal** person.

## Benefits

- **Global trade** gateway: **IEC** is the usual licence layer for **cross-**border **goods** and **EXIM-****linked** **services** in a firm context
- **Lifetime** number (with **annual** **DGFT** **confirm**)
- **Nil** government fee for issue and for standard modify / **annual** update
- **Single** code for all products and ports for that entity
- **Customs** and **bank** process **IEC** on **BoE**, **SB**, and **FEMA-****style** remittances
- **Schemes** and **RCMC** are usually built on a live **IEC**
- **Marketplaces** and **B2B** **credibility** in **EXIM** often expect **IEC**

## Limitations

- **Annual** **April**–**June** update, or **deactivation**
- **PAN** errors flow into **IEC**; **PAN** first, then **IEC**
- **Restricted** or **prohibited** goods need **extra** licences, not only **IEC**
- **SCOMET** and similar need **further** **authorisation** beyond **IEC**
- **Misuse** / identity risk if data is not guarded, mitigated in part by **Aadhaar** / **DSC** controls in current portal design

## Process

1. Create a dgft.gov.in user. Verify e-mail and mobile.
2. Link Aadhaar for the authorised signatory as the portal requires.
3. Open Apply for IEC (Form ANF-2A or the current DGFT form). PAN-linked fields may auto-populate.
4. Complete entity name, address, partners or directors, bank account, and GST if registered.
5. Upload address proof, bank proof, and constitution documents in allowed formats and sizes.
6. Government fee is nil. Submit with Aadhaar OTP (or EVC or DSC as the rule in force requires).
7. Note the application reference (ARN) and track status.
8. On approval, download the e-IEC. Processing is often the same day for clean data; else about two to five working days.
9. After IEC: register on ICEGATE, obtain RCMC if needed, file LUT for zero-rated goods exports, align AD bank, and add STPI if you are in software and SOFTEX rules apply.

## Use cases (short)

1. Garment exporter: IEC before international fairs, RCMC, ICEGATE, and LUT.
2. E-commerce global selling: marketplaces require IEC for Indian sellers.
3. Import of raw materials: CHA needs your IEC to file Bill of Entry.
4. Software services: IEC for SOFTEX above threshold, FEMA, and GST export reporting on advice.
5. Trust importing donated equipment: IEC as importer, plus exemption route if notified.

## Frequently asked questions

**Freelancer, only services, need IEC?** Light structures sometimes operate without; registered businesses doing formal exports, higher SOFTEX, or FEMA-linked flows usually need an IEC. Your CA should align entity type and bank path.

**Name mismatch reject?** Match DGFT form text exactly to PAN. Fix PAN with IT if it differs from RoC, then reapply.

**Multiple IECs for divisions?** One PAN, one IEC for one company. Separate legal persons get separate IECs.

**No annual update for years?** Check status on DGFT; if deactivated, complete overdue annual update, often with back years as the portal allows.

**Personal import of a collector bike?** Often personal or baggage rules apply; if commercial later, IEC may be needed. Confirm HS code and intent with a CHA.
